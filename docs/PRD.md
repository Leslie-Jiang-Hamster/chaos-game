# PRD: 混沌生还者

## 1. 产品定义

- 类型：单人、终端文字、叙事博弈、生存淘汰、AI Agent 模拟
- 玩家结构：1 名玩家 + 31 名选手 agent + 广播 agent + 环境 agent
- 赛制：5 轮，人数 `32 -> 16 -> 8 -> 4 -> 2 -> 1`
- 目标：玩家活到最后并独得奖金
- 失败：任意轮被淘汰死亡

## 2. 核心体验

- 规则不复杂，难点在读人、误导、结盟、反背刺
- 每个角色都有身世、动机、底线和可利用点
- 每轮游戏前后都有社交阶段
- 结果由规则、记忆和临场判断共同决定

## 3. 文本风格

- 整体文本基调偏细节描写、环境渲染和文学性表达，不走纯功能说明风
- 场景文本要先让玩家“身处其中”，再给规则信息
- 环境 agent 的描述应强调光线、气味、声音、空间压迫感、人物姿态等可感知细节
- 广播 agent 的语言应简短、冷静、制度化，与环境描写和角色发言形成明显反差
- 角色发言不追求平均信息密度，应允许停顿、试探、含混、情绪和潜台词
- 规则说明可以清楚，但不应把整体文本写成操作手册
- 玩家在终端里看到的文字，优先是叙事和氛围，其次才是系统提示

## 4. MVP 边界

### 必须有

- 完整 5 轮流程
- 31 名可行动选手 agent
- 广播 agent 与环境 agent
- 纯终端文字交互
- 社交阶段
- 记忆系统

## 5. 回合结构

每轮固定阶段：

1. 公共生活阶段
2. 赛前情报阶段
3. 游戏执行阶段
4. 结算淘汰阶段
5. 余波阶段

非游戏阶段重点：
- 了解彼此
- 交换信息
- 拉拢和误导
- 制造信任
- 埋下背叛

## 6. Agent 结构

### 选手 agent

每名选手至少包含：
- 公开身份
- 隐藏目标
- 性格与风险偏好
- 底线
- 私有记忆
- 最近对话窗口

### 广播 agent

职责：
- 宣读规则
- 宣布作息
- 推进阶段
- 公布结果
- 在混乱时控场

### 环境 agent

职责：
- 描述当前公共环境和时间
- 告知当前存活人物
- 响应通过 `speak(...)` 发起的环境查询
- 负责终端中的现场感

## 7. 记忆与判断

### 原则

- “关系”不是独立数值系统，而是记忆中的长期判断和倾向
- 某人是否可信、是否可利用、是否令人恐惧、是否值得报复，统一保存在记忆里
- agent 不显式调用记忆接口，而是在生成决策时潜移默化使用记忆

### 记忆类型

- 事件记忆
- 对话记忆
- 推断记忆
- 情绪记忆
- 倾向记忆

系统必须记住：
- 谁承诺过什么
- 谁骗过谁
- 谁救过谁
- 谁公开羞辱过谁
- 谁在关键轮投了谁
- 谁让谁产生好感、怀疑、厌恶或恐惧

### Agent 可见输入

每个选手 agent 仅可见：
- 自己的背景设定
- 自己的记忆
- 一定长度的最近对话，包括与人的对话和与环境 agent 的查询往来
- 全局信息，如当前游戏、规则、阶段、当前存活对象、可调用接口等

## 8. 五轮游戏

### 第 1 轮：诱饵均值

- 人数：`32 -> 16`
- 规则：
  - 所有人先公开发言
  - 然后各自秘密提交 `0-100` 的数字
  - 目标数 = 全体真实平均值的 `1/2`
  - 距离目标最近的 16 人存活
- 重点：公开共识可以被故意污染

### 第 2 轮：镜桥同行

- 人数：`16 -> 8`
- 规则：
  - 先配对
  - 每组共同走 6 段桥
  - 每段决定谁先走，再决定走哪边
  - 双方掌握不同桥面信息
  - 组内最终只活 1 人
- 重点：合作信息共享和转嫁风险同时存在

### 第 3 轮：四门筹码

- 人数：`8 -> 4`
- 规则：
  - 4 扇门，每扇门只活 1 人
  - 每人持 10 枚筹码，必须分配到 4 扇门
  - 每门最高者获胜
  - 一人最多赢 1 门
- 重点：口头分门协议和临场偷门背刺

### 第 4 轮：钥匙与刀

- 人数：`4 -> 2`
- 规则：
  - 每人秘密投出 1 把钥匙和 1 把刀
  - 钥匙代表保，刀代表杀
  - 分数高的 2 人存活
- 重点：纯拉票、纯追责

### 第 5 轮：终局审问

- 人数：`2 -> 1`
- 规则：
  - 双方先固定一个骰子点数 `1-6`
  - 进入审问轮
  - 每人每轮可提问，双方通过回答互相试探
  - 每人对自己给出的 4 个回答必须满足 `2 真 2 假`
  - 任一方可猜对方点数
  - 猜错直接死
  - 双方都不猜则下一轮继续，点数不变
- 重点：读人，不是算题

## 9. 终端交互

- 玩家通过命令或选项操作
- 所有规则、角色、公共环境、结果都以文字呈现
- 环境查询统一映射为对环境 agent 的 `speak(...)`
- 广播文本与角色发言必须明显区分
- 开场应先通过环境 agent 建立陌生感、压迫感和空间感，再逐步进入规则宣读和正式游戏

## 10. 关键风险

- 角色说话同质化
- agent 使用不该知道的信息
- 社交阶段太空，实际无决策价值
- 游戏规则可算但不好玩
- 角色太多，玩家记不住
