# 回合结构设计

## 1. 全局规则

### 非游戏阶段基础动作

```text
speak(target_scope, text)
end_phase()
```

规则：
- 非游戏阶段只开放这两个动作
- 角色可多次 `speak(...)`
- 角色结束时调用 `end_phase()`
- 倒计时归零时，系统强制所有未结束角色 `end_phase()`

### 游戏阶段

游戏阶段动作面：

```text
speak(...)
end_phase()
...本轮专属动作...
```

## 2. 阶段状态机

```text
round_start
-> daily_life
-> pre_game
-> game
-> result
-> aftermath
-> next_round
```

## 4. 第 1 轮：诱饵均值

### 赛前情报阶段

```text
available_actions = [
  "speak",
  "end_phase"
]
```

### 游戏执行阶段

```text
available_actions = [
  "speak",
  "choose_number",
  "end_phase"
]
```

规则：
- 先允许公开发言
- 然后每人秘密执行一次 `choose_number(value)`
- 结算目标数 = 全体真实平均值的 `1/2`
- 前 16 名存活

## 5. 第 2 轮：镜桥同行

### 赛前情报阶段

```text
available_actions = [
  "speak",
  "end_phase"
]
```

### 游戏执行阶段

```text
available_actions = [
  "speak",
  "choose_partner",
  "choose_order",
  "choose_path",
  "end_phase"
]
```

规则：
- 先锁定搭档：`choose_partner(target_id)`
- 每段先决定谁先走：`choose_order(mode)`
- 再选择路径：`choose_path(side)`
- 双方掌握不同桥面信息
- 组内只活 1 人

## 6. 第 3 轮：四门筹码

### 赛前情报阶段

```text
available_actions = [
  "speak",
  "end_phase"
]
```

### 游戏执行阶段

```text
available_actions = [
  "speak",
  "assign_tokens",
  "end_phase"
]
```

规则：
- 每人一次性提交 `assign_tokens(distribution)`
- 分配必须覆盖 4 扇门
- 每门最高者获胜
- 一人最多赢 1 门

## 7. 第 4 轮：钥匙与刀

### 赛前情报阶段

```text
available_actions = [
  "speak",
  "end_phase"
]
```

### 游戏执行阶段

```text
available_actions = [
  "speak",
  "give_key_to",
  "give_blade_to",
  "end_phase"
]
```

规则：
- 每人秘密交出 1 把钥匙和 1 把刀
- 钥匙保人，刀杀人
- 分数前 2 名存活

## 8. 第 5 轮：终局审问

### 预备阶段

```text
available_actions = [
  "choose_die_face"
]
```

### 审问阶段

```text
available_actions = [
  "speak",
  "end_phase"
]
```

规则：
- 双方先固定点数：`choose_die_face(value)`
- 审问通过普通 `speak(...)` 完成，提问或回答由发言内容和当前轮次语境决定
- 每人对自己的 4 个回答必须满足 `2 真 2 假`
- 双方都可多轮审问，点数不变

### 猜测阶段

```text
available_actions = [
  "guess_die_face",
  "pass_guess",
  "end_phase"
]
```

规则：
- 猜错直接死
- 双方都 `pass_guess()` 则进入下一轮审问

## 9. 玩家输入映射

玩家在 Web 聊天界面的输入与按钮动作，系统内部统一映射到动作接口。

示例：

```text
在私聊 014 窗口输入：先别公开站队
在私聊 032 窗口输入：你的点数是奇数吗
在公共大厅输入：别信他
```

对应：

```text
speak({ type: "single", ids: ["014"] }, "先别公开站队")
speak({ type: "single", ids: ["032"] }, "你的点数是奇数吗")
speak({ type: "public" }, "别信他")
```

说明：
- 公共大厅输入默认映射为 `speak({ type: "public" }, text)`
- 私聊窗口输入默认映射为 `speak({ type: "single", ids: [...] }, text)`
- 关键环境变化由系统消息直接写入公共时间线，不走玩家动作

## 10. 引擎职责

- 生成 `available_actions`
- 维护阶段时钟
- 处理 `end_phase()`
- 校验游戏动作参数
- 结算本轮结果
- 把 `speak(...)` 按 `public`、`single` 分发给对应听者
- 在关键时间点插入公共环境系统消息与广播主持人消息
